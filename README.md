## 📌 Sobre o Projeto

O **WellMe – Care Plus** é uma solução digital voltada ao acompanhamento da saúde e bem-estar dos usuários, permitindo o monitoramento de hábitos saudáveis, metas pessoais e indicadores relacionados à qualidade de vida.

O projeto foi desenvolvido utilizando práticas de metodologias ágeis, arquitetura corporativa e gestão de requisitos através do Azure DevOps, garantindo organização, rastreabilidade e controle do ciclo de desenvolvimento.

---

## 🎯 Objetivos

- Incentivar hábitos saudáveis.
- Permitir o acompanhamento de metas de saúde.
- Facilitar o monitoramento do progresso do usuário.
- Centralizar informações relacionadas ao bem-estar.
- Aplicar boas práticas de Engenharia de Software e Quality Assurance.

## 📋 Funcionalidades Principais

### Gestão de Hábitos

- Cadastro de hábitos saudáveis.
- Registro diário de atividades.
- Acompanhamento de progresso.
- Definição de metas personalizadas.

### Monitoramento de Saúde

- Controle de metas pessoais.
- Histórico de atividades.
- Visualização da evolução dos hábitos.
- Acompanhamento de indicadores de bem-estar.

### Engajamento do Usuário

- Gamificação
- Notificações e lembretes.
- Registro de conquistas.
- Incentivo à manutenção de hábitos saudáveis.

---

## 🧪 Estratégia de Qualidade (QA)

O projeto contempla atividades de Quality Assurance (QA), incluindo:

- Planejamento de testes.
- Criação de Test Cases.
- Validação de requisitos.
- Testes baseados em User Stories.
- Rastreabilidade entre PBIs e Test Cases.
- Gestão de testes através do Azure DevOps.

### Tipos de Teste

- Testes Funcionais
- Testes de Integração
- Testes de Aceitação
- Testes de Usabilidade

---

## 📂 Estrutura do Backlog

O gerenciamento do produto foi realizado utilizando a seguinte estrutura:

```text
Epic
 ├── Feature
 │     ├── Product Backlog Item (PBI)
 │     │      ├── Task
 │     │      └── Test Case
```

Essa organização garante rastreabilidade completa entre requisitos, desenvolvimento e testes.
